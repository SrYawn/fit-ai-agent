# 项目概览

## 目标
本仓库实现了一个围绕多 Agent 工作流构建的健身 AI 助手。系统的核心能力是理解用户的健身相关请求，将其路由到合适的处理流程，结合知识库或用户数据进行支撑，并生成训练计划、动作指导或闲聊回复等结果。

## 主要模块
- 根应用：Spring Boot API，负责暴露工作流接口并协调各类 Agent。
- `fitness-db-mcp-server/`：基于 MySQL 的 MCP 服务，提供用户画像、伤病和训练记录查询能力。
- `yu-image-search-mcp-server/`：图片搜索 MCP 服务。
- `docker/`：本地基础设施，包括 MySQL、Elasticsearch、Kibana 和 Adminer。

## 主要运行链路
1. 接收用户输入和可选的 `userId`。
2. 执行意图识别。
3. 路由到计划生成、动作指导或闲聊流程。
4. 若进入计划生成流程，先生成用户画像，再生成训练计划。
5. 普通接口返回结构化 JSON；流式接口返回 token 级 SSE，并在 payload 中附带节点元数据。

## 当前重点
- 保证用户画像尽可能基于真实数据落地。
- 缺少关键信息时优先明确降级，不伪造个性化内容。
- 保持工作流进度和外部依赖失败的可观测性。
