# 故障与排查流程

## 当依赖失败时
1. 明确是哪个依赖出问题。
2. 根据 `project/environments.md` 核对启动顺序和账号配置。
3. 将实际报错记录到当前变更目录中。
4. 判断本次场景应快速失败、显式降级，还是直接阻断。
5. 将最终处理方式记录到 `design.md` 和 `implementation.md`。

## 常见场景
- JDBC 连接失败：
  - 检查 Docker 中的 MySQL 是否运行。
  - 检查 `mcp-servers.json` 里的数据库账号配置。
  - 检查 DB MCP 的 `application.yml`。
- SSE 行为不符合预期：
  - 对比实际事件流和预期工作流阶段。
  - 将最终事件模型记录到当前变更目录。

## 文档职责
如果根因是启动前置条件缺失，或开发文档中的配置说明不一致，必须同步更新对应的项目级文档和开发入口说明。
