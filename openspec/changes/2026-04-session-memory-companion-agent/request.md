# 需求：多轮会话支持 + 训练状态与情绪双感知陪伴激励助手

## 背景
1. 当前每次请求无状态，BaseAgent.cleanup() 清空 messageList，连续对话无上下文
2. 闲聊路由（ChatAgent）功能单薄，缺乏训练感知和情绪激励能力

## 需求点
1. **多轮会话支持**：session 级上下文记忆（不跨会话持久化），后端自动生成 sessionId
2. **陪伴激励助手**：替换 ChatAgent，实现训练状态—情绪双感知的个性化陪伴激励模型
   - 分层 Prompt 工程：角色约束 / 情境感知 / 激励策略 / 安全控制
   - RAG 激励心理学知识库
   - MCP 训练数据感知 + LLM 情绪检测

## 验收条件
- 连续对话能引用之前的上下文
- 意图承接正确（"加上饮食建议" 仍路由到 plan_generation）
- 陪伴激励回复包含共情 + 策略，而非简单闲聊
- 训练数据感知：回复引用真实训练记录
- 安全边界：疼痛/伤病建议就医
- SSE 流首个 metadata 事件包含 sessionId
