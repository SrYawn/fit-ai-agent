# 决策记录

## 决策 1
- 背景：此前大量开发规范和上下文只放在 `AGENTS.md` 中，不利于项目知识沉淀和需求过程留痕。
- 决策：引入 `openspec/` 作为长期的结构化事实来源，并将 `AGENTS.md` 收敛为入口路由和约束文件。
- 影响：后续 Agent 应按照统一的 spec-first 流程进行开发和记录。

## 决策 2
- 背景：用户画像链路从 graph 传入的是 `userId`，但原先主要依赖按用户名查询的工具。
- 决策：新增按 `userId` 查询画像的 DB 工具，并明确禁止 Agent 进行用户名猜测。
- 影响：减少无效重试和错误推理，降低伪造用户数据的风险。
